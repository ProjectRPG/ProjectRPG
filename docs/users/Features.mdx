# Features

Project RPG enhances Minecraft with RPG elements through addons. As a hub mod, it doesn't add features itself but enables addons to introduce mechanics like abilities, progression, and gating. This page explains what these features do for players.

## Abilities
Abilities are special actions players can activate, such as buffs, attacks, or utilities.
- **How it works**: Addons define abilities with conditions (e.g., cooldowns, required progress). Players activate them via commands, items, or events.
- **Examples**: A "Fireball" ability that shoots a projectile after reaching level 5 in magic progression.
- **Player benefits**: Adds depth to gameplay, allowing unique playstyles.

## Progression
Progression tracks player advancement, like experience or skill levels.
- **How it works**: Addons implement systems with containers (e.g., "mining", "combat"). Progress is awarded from events like mining or fighting.
- **Examples**: Gaining XP from breaking blocks, unlocking new abilities at milestones.
- **Player benefits**: Motivates exploration and skill-building.

## Gating
Gating restricts access to content based on progress.
- **How it works**: Prevents actions (e.g., using items, progressing in skills) if requirements aren't met.
- **Examples**: Blocking a sword ability until combat level 10.
- **Player benefits**: Creates progression gates, encouraging balanced advancement.

## Features
Features allow other mods to integrate with the RPG system for their own purposes.
- **How it works**: Mods can register features to track player actions or modify gameplay based on RPG state, without affecting core progression or abilities.
- **Examples**: A factions mod using features to strengthen land protection based on player behaviors, or a story mod tracking unique actions for narrative progression.
- **Player benefits**: Enables richer, interconnected mod experiences.

## Events
Events trigger RPG mechanics in response to in-game actions.
- **How it works**: Addons register listeners for events like block breaking. Events process gating, features, abilities, and progression.
- **Examples**: Mining ore awards progression and possibly activates a feature.
- **Player benefits**: Makes gameplay feel responsive and RPG-like.

## Party System
Supports group-based mechanics.
- **How it works**: Addons can implement parties for shared progress or abilities.
- **Examples**: Party members share XP from kills.
- **Player benefits**: Enhances multiplayer RPG experiences.
